## 1. 제품 비전

**한 줄 요약**: 자연어로 건물을 설명하면, AI가 3D 모델을 만들고, 에너지 시뮬레이션을 실행하고, 최적 절감 전략을 추천하는 SaaS 플랫폼.

### 핵심 가치 제안

| 기존 방식 | EnergySimPlatform |
|-----------|-------------------|
| EnergyPlus IDF 수동 작성 (전문가 전용) | 자연어 입력 → 자동 생성 |
| Blender/SketchUp 별도 학습 필요 | AI가 3D 모델 생성, 웹에서 수정 |
| 시뮬레이션 결과 CSV/텍스트 분석 | 인터랙티브 대시보드 |
| EMS 전략 수동 적용/비교 | 원클릭 M0~M8 자동 비교 |
| 수주~수개월 소요 | 수분~수시간 |

### 경쟁 포지셔닝

```
               전문성 높음
                  │
    EnergyPlus    │   DesignBuilder
    (CLI/IDF)     │   (전문 GUI)
                  │
    ─────────────-┼──────────────── 자동화 높음
                  │
    OpenStudio    │   ★ EnergySimPlatform
    (SDK)         │   (NL → 자동 → 대시보드)
                  │
               전문성 낮음
```

## 2. 사용자 페르소나

### P1: 건물주 (비전문가)
- "내 건물 에너지 비용을 줄이고 싶다"
- 기대: 건물 사진/도면 업로드 → 절감 전략 추천 → ROI 확인
- 전문 용어 이해 불가, 시각적/직관적 결과 필요

### P2: 시설관리자 (준전문가)
- "HVAC 운영 조건을 바꾸면 얼마나 절약되나 비교하고 싶다"
- 기대: 운영 조건(스케줄, 설정온도) 조정 → 즉시 비교
- 기본 HVAC 용어 이해, 상세 설정 가능

### P3: 에너지 컨설턴트 (전문가)
- "고객에게 에너지 절감 보고서를 빠르게 제공하고 싶다"
- 기대: 정밀한 파라미터 제어, 커스텀 EMS, PDF 보고서 자동 생성
- IDF 직접 편집 가능, 고급 분석 필요

## 4. 모듈별 상세 스펙

### 4.1 모듈 ① — 건물 생성 (Building Creator)

#### 4.1.1 입력 방식 (4가지)

**A. 자연어 입력 (Primary)**
```
사용자: "서울 강남에 있는 12층 유리 커튼월 오피스 빌딩,
        지하 2층 주차장, 1층 로비, 연면적 약 46,000㎡"

AI 응답: 건물 파라미터 추출 → 3D 모델 생성 → 웹 뷰어에 표시
        → "이 건물이 맞나요? 수정하시겠습니까?"
```

- LLM이 자연어에서 건물 파라미터를 추출
- 파라미터: 용도, 층수, 연면적, 구조, 외피, 창호, 지역 등
- 추출 결과를 사용자에게 확인 (카드형 요약)
- 누락/모호한 정보는 AI가 합리적 기본값 제안 + 사용자 확인

**B. 사진/도면 업로드**
```
입력: 건물 외관 사진, 평면도 CAD/PDF
처리: Vision AI → 건물 특성 추출 → 파라미터화 → 3D 생성
출력: 3D 모델 + 추출된 파라미터 (수정 가능)
```

**C. 기존 3D 모델 임포트**
```
지원 형식: .blend, .obj, .fbx, .ifc (BIM), .gbXML
처리: 형상 분석 → Zone 자동 분할 → 파라미터 추출
```

**D. 건물 유형 템플릿**
```
프리셋: DOE Reference Buildings 기반
- Large Office (46,320 m², 12F, VAV+Chiller)
- Medium Office (4,982 m², 3F, VRF)
- Small Office (511 m², 1F, PSZ-HP)
- Standalone Retail (2,294 m², 1F, PSZ-AC)
- Primary School (6,871 m², 1F, VAV)
- Hospital (22,422 m², 5F, Chiller+VAV)
- Hotel (신규)
- Apartment (신규)
사용자가 파라미터 조정 (층수, 면적, 창호비 등)
```

#### 4.1.2 3D 모델 생성 파이프라인

```
자연어/사진/파라미터
        │
        ▼
┌─────────────────────┐
│ Building Parameter   │  ← 중간 표현 (JSON Schema)
│ Schema (BPS)        │     건물의 모든 물리적 특성을 구조화
└─────────────────────┘
        │
        ▼
┌─────────────────────┐
│ Blender Python API  │  ← 서버사이드 Blender (headless)
│ (bpy scripting)     │     BPS → Blender 3D 모델 자동 생성
└─────────────────────┘
        │
        ▼
┌─────────────────────┐
│ glTF/glb Export     │  ← 웹 3D 뷰어용 경량 포맷
│ + IDF Export        │     + EnergyPlus IDF 동시 생성
└─────────────────────┘
```

```json
{
  "project_id": "uuid",
  "name": "강남 오피스 빌딩",
  "location": {
    "city": "Seoul",
    "district": "강남구",
    "latitude": 37.4979,
    "longitude": 127.0276,
    "climate_zone": "4A"
  },
  "geometry": {
    "building_type": "large_office",
    "num_floors_above": 12,
    "num_floors_below": 2,
    "total_floor_area_m2": 46320,
    "floor_to_floor_height_m": 3.96,
    "aspect_ratio": 1.5,
    "footprint_shape": "rectangle",
    "wwr": { "north": 0.4, "south": 0.6, "east": 0.4, "west": 0.4 }
  },
  "envelope": {
    "wall_type": "curtain_wall",

    {
      "name": "lobby",

      "usage": "lobby",
      "occupancy_density": 0.1
    }
  ],
  "hvac": {
    "system_type": "vav_chiller_boiler",
    "cooling_source": "chiller",
    "heating_source": "boiler",
    "num_chillers": 2,
    "num_boilers": 2,
    "has_economizer": false,
    "has_data_center": true
  },
  "schedules": {
```

**핵심**: 이 BPS가 전체 시스템의 SSOT (Single Source of Truth).
사용자가 수정하는 것도 이 BPS, 3D 생성도 이 BPS, IDF 생성도 이 BPS 기반.

#### 4.1.4 웹 3D 뷰어

- **기술**: React Three Fiber (Three.js + React)
- **기능**:
  - 건물 3D 모델 회전/확대/축소
  - 층별/존별 클릭 → 상세 정보 패널
  - 간단한 편집: 층수 변경, WWR 슬라이더, 형상 비율 조정
  - 외피/창호 색상으로 열적 성능 시각화 (히트맵)
- **수정 → 반영**: 뷰어에서 파라미터 수정 → BPS 업데이트 → 3D 재생성 (실시간)

### 4.2 모듈 ② — 건물 설정 (Building Configuration)

단계별 위저드 형태로 사용자가 시뮬레이션 조건을 설정.

#### 4.2.1 설정 단계

```
Step 1: 위치 & 기후
├── 지도에서 위치 선택 (또는 자동 감지)
├── EPW 파일 자동 매칭
├── 기후대 표시 (ASHRAE Climate Zone)
└── 설계 외기 조건 표시

Step 2: 운영 조건
├── 용도별 프리셋 (오피스, 학교, 병원 등)
├── 근무일/휴일 캘린더 (한국 공휴일 자동 반영)
├── 운영 시간 (시작/종료)
├── 토요일 운영 여부
└── 재실 밀도

Step 3: HVAC 설비
├── 설비 유형 추천 (건물 규모 기반)
│   ├── Large: VAV + Chiller + Boiler
│   ├── Medium: VRF
│   └── Small: PSZ-HP / PSZ-AC
├── 설비 용량 (autosize 또는 수동)
├── COP/효율 설정
└── 보조 설비 (환기, 가습 등)

Step 4: 설정온도
├── 프리셋: ASHRAE 55 기반 권장값
├── 난방 설정온도 (재실/비재실)
├── 냉방 설정온도 (재실/비재실)
└── PMV 목표값 (0.5 / 0.7 선택)

Step 5: 검토 & 확인
├── 전체 설정 요약 카드
├── 경고/권고 표시 (비합리적 설정 감지)
├── "시뮬레이션 시작" 버튼
└── 예상 소요 시간 표시
```

#### 4.2.2 스마트 기본값

각 설정에 AI 추천 기본값 제공:
- 건물 유형 + 지역 기반으로 최적 기본값 자동 설정
- 사용자는 기본값 그대로 "다음" 누르거나, 수정 가능
- P1(비전문가)은 대부분 기본값 사용 → 빠른 진행
- P3(전문가)는 모든 값 커스터마이징

### 4.3 모듈 ③ — Baseline 시뮬레이션 (Simulation Engine)

#### 4.3.1 BPS → IDF 변환

```
BPS (JSON)
    │
    ▼
┌─────────────────────┐
│ IDF Generator       │
│ (Python)            │
│                     │
│ 1. 기하학 변환      │  ← Zone, Surface, Window 객체 생성
│ 2. 재료/구조 적용   │  ← 외피 U-value → Construction 매핑
│ 3. HVAC 템플릿 적용 │  ← 건물유형별 HVAC 시스템 자동 구성
│ 4. 스케줄 생성      │  ← 운영조건 → Schedule:Compact
│ 5. 출력 변수 설정   │  ← 표준 미터 + 커스텀 출력
│ 6. 검증             │  ← 물리적 타당성 + 참조 무결성
└─────────────────────┘
    │
    ▼
model.idf + metadata.json (SHA256)
```

#### 4.3.2 시뮬레이션 실행

- **엔진**: EnergyPlus 24.1+ (서버사이드)
- **실행 환경**:
  - 로컬: 단일 빌딩 테스트
  - 클라우드: GCP Cloud Batch / AWS Batch (대규모 배치)
- **진행 상황**: WebSocket으로 실시간 진행률 전달
  - "IDF 생성 중..." → "시뮬레이션 실행 중 (40%)..." → "결과 처리 중..."
- **에러 처리**: Severe Error 감지 → 사용자에게 원인 설명 + 수정 제안

#### 4.3.3 결과 수집

```
EnergyPlus 출력
    │
    ├── eplusout.csv     → 시계열 에너지 데이터
    ├── eplusmtr.csv     → 미터 데이터
    ├── eplustbl.htm     → 요약 테이블
    └── eplusout.err     → 에러/경고 로그
    │
    ▼
┌─────────────────────┐
│ Result Processor     │
│                     │
│ - 시계열 → DB 저장  │
│ - EUI 계산          │
│ - 피크 전력 추출    │
│ - 존별 온도 통계    │
│ - Unmet Hours 계산  │
└─────────────────────┘
    │
    ▼
결과 DB (PostgreSQL / TimescaleDB)
```

### 4.4 모듈 ④ — EMS 전략 비교 (Strategy Comparison)

#### 4.4.1 자동 EMS 적용

Baseline 완료 후, 사용자에게 질문:
```
"에너지 절감 전략을 자동으로 비교하시겠습니까?"
  [전체 비교 (M0~M8)]  [선택 비교]  [건너뛰기]
```

**전체 비교 선택 시**:
```
BASELINE 결과
    │
    ▼
┌──────────────────────────────┐
│ 공정 조건 자동 적용           │
│                              │
│ - 설비 크기 고정 (Baseline)  │
│ - 동일 기간/기후/타임스텝    │
│ - 동일 재실 스케줄           │
│ - 동일 Setback 온도          │
│ - DataCenter 부하 고정       │
└──────────────────────────────┘
    │
    ▼
M0~M8 IDF 9개 자동 생성 → 병렬 시뮬레이션
```

#### 4.4.2 EMS 전략 설명 (사용자 UI)

| 전략 | 사용자용 이름 | 아이콘 | 한 줄 설명 |
|------|-------------|--------|-----------|
| M0 | 야간 정지 | 🌙 | 퇴근 후 HVAC 자동 정지 |
| M1 | 스마트 시작 | ⏰ | 기온에 따라 출근 전 예열/예냉 |
| M2 | 외기 냉방 | 🌬️ | 외기가 시원할 때 자연 환기 활용 |
| M3 | 냉동기 단계제어 | ❄️ | 부하에 맞게 냉동기 대수 조절 |
| M4 | 쾌적 최적화 (보통) | 😊 | PMV 0.5 기준 설정온도 자동 조정 |
| M5 | 쾌적 최적화 (절약) | 💰 | PMV 0.7 기준 더 넓은 허용 범위 |
| M6 | 설비 통합제어 | 🔧 | M2+M3 복합 적용 |
| M7 | 통합+쾌적(보통) | 🏆 | M6+M4 최적 조합 |
| M8 | 통합+쾌적(절약) | 🥇 | M6+M5 최대 절감 |

#### 4.4.3 건물 유형별 전략 적용 가능성 (자동 필터링)

시스템이 건물 HVAC 유형에 따라 적용 불가 전략을 자동 비활성화:
```
Large Office (VAV+Chiller): M0~M8 전체 적용 가능
Medium Office (VRF):        M0, M4, M5만 적용 (M2, M3, M6~M8 비활성)
Small Office (PSZ-HP):      M0, M2, M4, M5 적용 (M3 비활성 → M6=M2)
```

## 7. 논의 필요 항목

아래 항목들은 스펙 확정 전 사용자와 논의가 필요합니다.

### DISCUSS-001: 자연어 → BPS 정확도 요구 수준

자연어 입력에서 건물 파라미터를 추출할 때, 누락 정보 처리 방식:
- **A안**: AI가 합리적 기본값으로 자동 채움 + 사후 확인
- **B안**: 누락 항목마다 추가 질문 (대화형)
- **C안**: 필수 항목만 확인, 나머지 기본값

### DISCUSS-002: Blender 서버 비용 및 대안

Blender headless 서버는 GPU 필요할 수 있음:
- 단순 형상 생성은 CPU만으로 가능
- 사실적 렌더링이 필요하면 GPU 필요
- 대안: Three.js로 프론트엔드에서 직접 생성 (Blender 불필요)

### DISCUSS-003: EnergyPlus 실행 환경

SaaS에서 EnergyPlus 실행 시:
- **A안**: 서버에서 직접 실행 (단순, 비용 높음)
- **B안**: 서버리스 (AWS Lambda / Cloud Run) — 콜드스타트 이슈
- **C안**: 사용자 로컬 실행 옵션 (비용 절감, 설치 필요)

### DISCUSS-004: 과금 모델

SaaS 과금 방식:
- **A안**: 시뮬레이션 건수당 과금
- **B안**: 월정액 구독 (건물 수 제한)
- **C안**: 프리미엄 (기본 무료 + 고급 기능 유료)

### DISCUSS-005: 초기 지원 기후/지역 범위

- **A안**: 한국 10개 도시만 (기존 EPW 활용)
- **B안**: 한국 전역 (TMY 데이터)
- **C안**: 글로벌 (EPW 자동 다운로드)

### DISCUSS-006: 프로젝트 이름

- EnergySimPlatform (가칭)
- 다른 제안?
